**Техническое задание на разработку CRM-системы для IT-академии "Hedgehog"**

**Версия:** 1.0
**Дата:** 27.12.2025
**Заказчик:** IT-академия "Hedgehog"

---

### **1. Введение**

#### **1.1. Назначение документа**

Настоящий документ определяет цели, требования, функциональность и ограничения CRM-системы, разрабатываемой для автоматизации бизнес-процессов IT-академии "Hedgehog". Система призвана объединить управление клиентами (студентами и их родителями), учебным процессом, продажами и коммуникациями в едином пространстве.

#### **1.2. Цель проекта**

Создать централизованную, удобную и эффективную CRM-систему, которая:

* Повысит конверсию лидов в оплачивающих студентов.
* Автоматизирует учет посещаемости и успеваемости.
* Упростит работу менеджеров по продажам и преподавателей.
* Обеспечит прозрачность учебного процесса для администрации и родителей.
* Улучшит качество обслуживания клиентов.

---

### **2. Общее описание системы**

CRM-система является веб-приложением с рольвой моделью доступа. Система состоит из взаимосвязанных модулей, обеспечивающих сквозную автоматизацию от первого контакта с клиентом до завершения им обучения.

**Ключевые сущности:**

* **Ученик:** дата рождения, имя, фамилия, отчество, телефон родителя, почта родителя, ФИО родителя
* **Курс:** название, стоимость курса, стоимость одного занятия, кол-во занятий
* **Группа:** 
* **Занятие (Урок):** Конкретное событие в расписании группы с темой, датой и временем.
* **Сделка:** Процесс продажи курса клиенту.
* **Задача:** Поручение для менеджера или преподавателя.

---

### **3. Роли пользователей и их возможности**

1. **Администратор:** Полный доступ ко всем функциям системы, управление пользователями, настройками, справочниками (курсы, цены), формирование отчетов.
2. **Менеджер (по продажам):**

   * Ведение воронки продаж (лиды -> сделки).
   * Управление клиентами (создание, редактирование, история взаимодействий).
   * Назначение задач себе и другим.
   * Формирование договоров и счетов
   * Просмотр расписания своих клиентов.
3. **Преподаватель:**

   * Просмотр своего расписания и списка групп.
   * **Ведение электронного журнала:** Отметка посещаемости на каждом занятии.
   * Ведение успеваемости (выставление оценок, комментариев за урок).
   * Просмотр истории пропусков и пройденных тем для конкретного ученика для организации отработок.
   * Закрепление домашних заданий или материалов за уроком.
4. **Родитель (Внешний пользователь):** Доступ через личный кабинет к:

   * Расписанию своего ребенка.
   * Истории посещаемости и успеваемости.
   * Учебным материалам и домашним заданиям.
   * Чат/уведомлениям от менеджера или преподавателя.

### **4. Функциональные требования (Модули системы)**

#### **4.1. Модуль "Клиенты"**

* **Хранение данных:** ФИО ученика, контакты (телефон, email), дата рождения, контакты родителей (ФИО, телефоны, email), ссылки на соцсети.
* **История взаимодействий:** Лог всех звонков, встреч, писем, платежей, комментариев менеджера. (Интеграция с телефонией и email).
* **Привязка к сделкам и группам:** Просмотр всех курсов, которые проходит или проходил ученик.
* **Теги и сегментация:** Возможность помечать клиентов тегами ("новичок", "активный", "пробный урок", "холодный") для фильтрации и рассылок.

#### **4.2. Модуль "Воронка продаж (Сделки)"**

* **Этапы воронки:** "Первичный контакт" -> "Пробный урок" -> "Выбор курса" -> "Оформление договора" -> "Оплата" -> "Учеба".
* **Управление сделкой:** Назначение ответственного менеджера, установка суммы, контроль сроков.
* **Автоматизация:** Создание задач при смене этапа (например, "отправить договор" при переходе на этап "Оформление").
* **Аналитика:** Воронка продаж, конверсия по менеджерам, средний чек.

#### **4.3. Модуль "Обучение" (Ядро системы)**

* **Управление курсами и модулями:** Создание иерархии: Курс -> Модули (2 шт.) -> Темы уроков (привязаны к каждому модулю).
* **Управление группами:** Создание группы, привязка к курсу и модулю, назначение преподавателя, установка расписания (дни, время, аудитория).
* **Календарь и расписание:** Единое расписание для всех групп. Возможность просмотра в разрезе преподавателя, группы, аудитории.
* **Электронный журнал (Критически важно):**
  * Для каждого **занятия** в расписании преподаватель отмечает: **посещаемость** (присутствовал/отсутствовал/опоздал) для каждого ученика.
  * Автоматическое отображение **темы урока** из программы модуля.
  * Возможность добавить **комментарий** по уроку или ученику.
  * **История пропусков:** Для каждого ученика в карточке формируется список пропущенных занятий с указанием даты и **темы пропущенного урока**. Это ключевое требование для организации отработок.
  * Прикрепление файлов (ДЗ, материалы) к уроку.

#### **4.4. Модуль "Задачи и события"**

* Создание задач с указанием исполнителя, дедлайна, приоритета и привязкой к клиенту/сделке.
* Напоминания о задачах (в системе, по email).
* Календарь событий (встречи, звонки, открытые уроки).

#### **4.5. Модуль "Аналитика и отчеты"**

* **Финансы:** Отчет по оплатам, долгам, выручке по курсам/менеджерам.
* **Учебный процесс:** Отчет по посещаемости по группам/преподавателям, успеваемости.
* **Продажи:** Отчет по эффективности менеджеров, источники заявок.
* **Загрузка:** Загруженность преподавателей и аудиторий.

#### **4.6. Модуль "Коммуникации" (Интеграции)**

* **Интеграция с телефонией (IP-телефония):** Совершение звонков из карточки клиента с автоматической записью разговора и созданием лога.
* **Интеграция с email и мессенджерами (Telegram, WhatsApp):** Отправка сообщений, шаблонные письма, история переписки в карточке клиента.
* **СМС/Email-рассылки:** Запланированные и триггерные рассылки (например, напоминание о занятии за час, отчет родителю в конце модуля).

#### **4.7. Личный кабинет для родителей/учеников**

* **Вид:** Веб-интерфейс.
* **Функции:** Расписание, журнал посещаемости и оценок, домашние задания, кнопка "Сообщить об отсутствии".

---

### **5. Нефункциональные требования**

* **Интерфейс:** Современный, интуитивно понятный, адаптивный.
* **Безопасность:** Разграничение прав доступа (RBAC), HTTPS, хеширование паролей, защита персональных данных (152-ФЗ).
* **Производительность:** Отклик интерфейса < 2 сек. при работе 50+ concurrent пользователей.
* **Надежность:** Резервное копирование данных ежедневно. Целевое время доступности 99.5%.
* **Масштабируемость:** Архитектура, позволяющая добавлять новые модули и увеличивать нагрузку.

### **6. Технический стек**

* **Бэкенд:** Python, fast-api
* **Фронтенд:** React
* **База данных:** PostgreSQL
* **Кеширование:** Redis.
* **Деплой:** Docker

---

**Приложение: Глоссарий**

* **Лид** - потенциальный клиент, проявивший интерес.
* **Сделка** - процесс продажи.
* **Модуль** - часть курса (в системе их всегда 2 на курс).
* **Отработка** - занятие для повторения пропущенного материала.
